# MODELO OFICIAL DAS AVALIAÇÕES FORMATIVAS

# Versionamento de Código com Git e GitHub

## Ensino Médio Técnico em Desenvolvimento de Sistemas

---

# 🎯 Objetivo

As Avaliações Formativas (AF) têm como objetivo verificar a compreensão dos conceitos estudados em cada módulo.

Não têm como foco apenas a memorização de comandos.

Buscam avaliar:

* Compreensão conceitual;
* Interpretação;
* Aplicação prática;
* Tomada de decisão;
* Resolução de problemas.

---

# 📊 Estrutura Padrão

Cada avaliação será composta por:

| Tipo de Questão     | Quantidade |
| ------------------- | ---------: |
| Múltipla Escolha    |          2 |
| Caixa de Seleção    |          2 |
| Verdadeiro ou Falso |          2 |
| Associação          |          2 |
| Situação-Problema   |          1 |
| Questão Desafio     |          1 |

Total:

10 questões

---

# 📖 BLOCO 1

# Múltipla Escolha

## Objetivo

Verificar a compreensão direta dos conceitos.

---

## Estrutura

* 5 alternativas;
* Apenas 1 correta;
* Distratores plausíveis.

---

## Exemplo

Qual comando é utilizado para registrar alterações no histórico do Git?

A) git push

B) git clone

C) git commit

D) git pull

E) git branch

---

# 📖 BLOCO 2

# Caixa de Seleção

## Objetivo

Avaliar a capacidade de identificar múltiplas informações corretas.

---

## Estrutura

* Entre 4 e 6 alternativas;
* Duas ou mais corretas.

---

## Exemplo

Selecione as funções desempenhadas pelo Git.

☐ Registrar alterações

☐ Armazenar histórico

☐ Editar imagens automaticamente

☐ Permitir trabalho colaborativo

☐ Recuperar versões anteriores

---

# 📖 BLOCO 3

# Verdadeiro ou Falso

## Objetivo

Verificar compreensão de conceitos fundamentais.

---

## Estrutura

Apresentar afirmações para classificação.

---

## Exemplo

( ) Git e GitHub são exatamente a mesma ferramenta.

( ) O Git pode ser utilizado sem internet.

( ) Um commit cria um registro no histórico do projeto.

( ) O GitHub hospeda repositórios Git.

---

# 📖 BLOCO 4

# Associação

## Objetivo

Avaliar a relação entre conceitos e funções.

---

## Estrutura

Relacionar elementos de duas colunas.

---

## Exemplo

### Coluna A

1. Git

2. GitHub

3. Commit

4. Branch

---

### Coluna B

( ) Plataforma de hospedagem de repositórios.

( ) Registro de alterações.

( ) Sistema de controle de versão.

( ) Linha paralela de desenvolvimento.

---

# 📖 BLOCO 5

# Situação-Problema

## Objetivo

Avaliar interpretação e aplicação do conhecimento.

---

## Estrutura

Apresentar um cenário próximo da realidade profissional.

---

## Exemplo

João trabalha em um projeto escolar.

Ele salvou cinco versões diferentes do mesmo arquivo:

trabalho.docx

trabalho_final.docx

trabalho_final_agora_vai.docx

trabalho_final_definitivo.docx

trabalho_final_definitivo2.docx

Explique como o Git poderia ajudar nessa situação.

---

# 📖 BLOCO 6

# Questão Desafio

## Objetivo

Estimular pensamento crítico.

---

## Estrutura

Questão aberta ou estudo de caso.

---

## Exemplo

Explique por que uma equipe de desenvolvimento que trabalha sem controle de versão corre riscos maiores do que uma equipe que utiliza Git.

Justifique sua resposta.

# 🎯 Princípios para Elaboração das Questões

As questões devem:

✅ Utilizar linguagem adequada ao Ensino Médio Técnico.

✅ Priorizar compreensão em vez de memorização.

✅ Utilizar exemplos reais.

✅ Relacionar teoria e prática.

✅ Utilizar situações do mercado de trabalho.

---

# 🚫 Evitar

❌ Questões excessivamente decorativas.

❌ Comandos sem contexto.

❌ Perguntas ambíguas.

❌ Pegadinhas.

❌ Terminologia excessivamente avançada para iniciantes.

---

# 🏆 Resultado Esperado

Ao final de cada avaliação, o estudante deverá demonstrar:

* Compreensão dos conceitos estudados;
* Capacidade de aplicar conhecimentos;
* Evolução progressiva ao longo da disciplina;
* Preparação para o próximo módulo.
