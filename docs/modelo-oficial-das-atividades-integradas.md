# MODELO OFICIAL DAS ATIVIDADES INTEGRADAS

# Versionamento de Código com Git e GitHub

## Ensino Médio Técnico em Desenvolvimento de Sistemas

---

# 🎯 Objetivo

As Atividades Integradas (AI) têm como objetivo permitir que o estudante aplique, em situações práticas, os conhecimentos desenvolvidos durante cada módulo.

Diferentemente das Avaliações Formativas, as AI priorizam:

* Aprendizagem prática;
* Resolução de problemas;
* Construção progressiva do Projeto Integrador;
* Uso real do Git e GitHub;
* Documentação técnica.

---

# 🚀 Relação com o Projeto Integrador

Todas as Atividades Integradas deverão contribuir diretamente para a construção do projeto:

## Meu Manual de Git e GitHub

Ao final de cada módulo o estudante deverá atualizar seu repositório pessoal.

Dessa forma, nenhuma atividade será descartada.

Tudo fará parte do produto final.

---

# 📋 Estrutura Oficial das AI

Todas as atividades seguirão o mesmo modelo.

---

## 1. Contextualização

Apresentação da situação-problema.

---

## 2. Objetivos

O que o estudante deverá demonstrar.

---

## 3. Requisitos

Lista das tarefas obrigatórias.

---

## 4. Evidências

O que deverá existir no GitHub ao final da atividade.

---

## 5. Entrega

Forma de envio.

---

## 6. Checklist

Itens para conferência antes da entrega.

# 📊 Critérios de Correção

Todas as AI utilizarão os mesmos critérios.

---

## Critério 1

### Organização

Peso: 2 pontos

Verificar:

* Estrutura do projeto;
* Nomeação adequada;
* Navegação.

---

## Critério 2

### Documentação

Peso: 2 pontos

Verificar:

* Clareza;
* Organização;
* Uso de Markdown.

---

## Critério 3

### Aplicação Técnica

Peso: 3 pontos

Verificar:

* Uso correto dos recursos estudados.

---

## Critério 4

### Cumprimento dos Requisitos

Peso: 3 pontos

Verificar:

* Atendimento aos itens solicitados.

# ✅ Checklist do Estudante

Antes de entregar qualquer AI, verificar:

□ Atualizei meu repositório.

□ Organizei os arquivos corretamente.

□ Revisei o README.

□ Realizei os commits necessários.

□ Sincronizei o GitHub.

□ Conferi os requisitos da atividade.

□ Testei os links existentes.
