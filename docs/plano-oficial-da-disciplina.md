# 2. JUSTIFICATIVA

O desenvolvimento de software moderno exige organização, rastreabilidade e colaboração.

Projetos profissionais raramente são desenvolvidos por apenas uma pessoa. Equipes de desenvolvimento trabalham simultaneamente em diferentes funcionalidades, correções e melhorias.

Nesse contexto, ferramentas de controle de versão tornaram-se essenciais para garantir:

* Histórico das alterações realizadas;
* Recuperação de versões anteriores;
* Trabalho colaborativo;
* Segurança do código-fonte;
* Documentação das decisões tomadas.

O Git é atualmente o sistema de controle de versão mais utilizado no mundo, enquanto o GitHub é uma das principais plataformas de hospedagem e colaboração em projetos de software.

Ao aprender Git e GitHub, o estudante desenvolve competências técnicas alinhadas às demandas do mercado de trabalho e fortalece habilidades relacionadas à cultura digital, comunicação, colaboração e resolução de problemas.
