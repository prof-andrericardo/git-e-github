# 🎯 Objetivos da Aula

Ao final da aula os estudantes deverão ser capazes de:

* Compreender o que é o GitHub.

* Identificar a importância do GitHub para o mercado de trabalho.
* Criar a estrutura inicial do Projeto Integrador.
* Produzir o primeiro README da disciplina.

# ⏱️ Cronograma da Aula (40 minutos)

## 1. Revisão da Aula 01 (5 minutos)

Perguntar:

* O que é versionamento?
* Por que o Git foi criado?
* Qual a diferença entre Git e GitHub?

---

## 2. Problematização (5 minutos)

Apresentar a situação:

> Dois estudantes possuem o mesmo conhecimento técnico.

Estudante A possui GitHub organizado.

Estudante B não possui GitHub.

Perguntar:

> Qual deles terá mais facilidade para demonstrar suas habilidades?

Permitir debate rápido.

---

## 3. Desenvolvimento Conceitual (10 minutos)

Explicar:

* O que é GitHub;
* O que é um repositório;
* O que é um perfil profissional;
* Como empresas utilizam GitHub.

Utilizar exemplos reais.

---

## 4. Mercado de Trabalho (5 minutos)

Apresentar profissões que utilizam GitHub:

* Desenvolvedor Front-end;
* Desenvolvedor Back-end;
* Desenvolvedor Full Stack;
* Desenvolvedor Mobile;
* DevOps;
* Cientista de Dados;
* Engenheiro de Software;
* Arquiteto de Software;
* Analista de Sistemas.

Relacionar com o curso técnico.

---

## 5. Projeto Integrador (5 minutos)

## 6. Mão na Massa (5 minutos)

```text

```

### Intervenção

