40 minutos

Meu Manual de Git e GitHub

---

# 🎯 Objetivos da Aula

Ao final da aula os estudantes deverão ser capazes de:

* Compreender o conceito de versionamento.
* Identificar problemas causados pela ausência de controle de versões.
* Entender a importância do histórico de alterações.
* Reconhecer a diferença entre Git e GitHub.
* Compreender por que o Git foi criado.
* Criar uma conta GitHub.

# 🎯 Conceito Central da Aula

O estudante deve compreender que:

> O Git não foi criado para ensinar comandos.

O Git foi criado para resolver problemas reais.

A ferramenta surge como consequência da necessidade.

Não devemos iniciar a aula ensinando comandos.

Primeiro devemos apresentar o problema.

Depois a solução.

# ⏱️ Cronograma da Aula (40 minutos)

## 1. Abertura (5 minutos)

Apresentar a seguinte situação:

```text id="wqctb5"
trabalho.docx

trabalho_final.docx

trabalho_final2.docx

trabalho_final_agora_vai.docx

trabalho_final_definitivo.docx
```

Perguntar:

> Qual é a versão correta?

Permitir que os alunos respondam.

---

## 2. Problematização (5 minutos)

Perguntar:

* Já perderam algum trabalho?
* Já sobrescreveram um arquivo?
* Já ficaram em dúvida sobre qual era a versão correta?

Estimular participação.

---

## 3. Desenvolvimento Conceitual (10 minutos)

Explicar:

* Versionamento;
* Histórico;
* Rastreabilidade;
* Recuperação;
* Colaboração.

Utilizar exemplos próximos da realidade dos alunos.

---

## 4. Git e Mercado de Trabalho (5 minutos)

* Microsoft;

---

## 5. Git x GitHub (5 minutos)

## 7. Encerramento (5 minutos)

Realizar revisão rápida.
