# 📢 Descrição – Aula 03

Nesta aula aprenderemos os fundamentos da documentação técnica utilizando Markdown, uma das linguagens mais utilizadas para criar READMEs, manuais e documentações em projetos de tecnologia. Também compreenderemos a importância da documentação para equipes de desenvolvimento e iniciaremos a construção estruturada do projeto integrador **Meu Manual de Git e GitHub**.

### 📚 Conteúdos abordados

* O que é Markdown;
* O que é README.md;
* Estrutura básica de documentação técnica;
* Títulos, listas, links e tabelas em Markdown;
* Organização de projetos;
* Conceito de commit;
* Atualização do Projeto Integrador.

### 🛠️ Atividade prática

* Criar e organizar o README.md do projeto;
* Criar a pasta da Aula 03;
* Produzir o resumo da aula;
* Estruturar a documentação inicial do Manual de Git e GitHub;
* Preparar a primeira mensagem de commit.

### 🎯 Objetivo

Compreender a importância da documentação técnica e aprender a utilizar Markdown para criar projetos organizados, claros e profissionais.

### 🚀 Entregáveis da Aula

✔ README.md estruturado.

✔ Pasta da Aula 03 criada.

✔ Resumo da Aula 03 produzido.

✔ Estrutura do Projeto Integrador atualizada.

✔ Primeira mensagem de commit definida.

### 💡 Importante

No mercado de trabalho, não basta apenas desenvolver software. Também é necessário documentar projetos de forma clara para facilitar a manutenção, o trabalho em equipe e a continuidade do desenvolvimento.
