# 🎯 Objetivos da Aula

Ao final da aula os estudantes deverão ser capazes de:

* Compreender o que é Markdown.
* Entender a importância da documentação técnica.
* Reconhecer a função do arquivo README.md.
* Utilizar elementos básicos do Markdown.
* Produzir um README organizado.
* Estruturar o Projeto Integrador.
* Compreender o conceito de commit.

# ⏱️ Cronograma da Aula (40 minutos)

## 1. Revisão Inicial (5 minutos)

Perguntar:

* O que é GitHub?
* O que é um repositório?
* Por que empresas utilizam GitHub?

---

## 2. Problematização (5 minutos)

Apresentar dois cenários:

### Projeto A

Com README completo.

---

### Projeto B

Sem README.

---

Perguntar:

> Qual projeto seria mais fácil de utilizar?

Estimular discussão.

---

## 3. Desenvolvimento Conceitual (10 minutos)

Explicar:

* O que é Markdown;
* O que é README;
* Onde são utilizados;
* Importância da documentação.

Demonstrar exemplos reais.

---

## 4. Demonstração Prática (5 minutos)

Mostrar:

```markdown
# Título

## Subtítulo

- Lista

[Link](https://github.com)
```

Explicar como o Markdown é convertido em documentação visual.

---

## 5. Projeto Integrador (5 minutos)

Orientar os estudantes a:

* Atualizar o README;
* Criar a pasta da Aula 03;
* Registrar o resumo da aula.

---

## 6. Conceito de Commit (5 minutos)

Introduzir:

> Commit é um registro permanente de alterações.

Utilizar a analogia:

```text
Fotografia
↓
Registro
↓
Histórico
```

Sem apresentar comandos ainda.

---

## 7. Encerramento (5 minutos)

Revisar:

* Markdown;
* README;
* Documentação;
* Projeto Integrador;
* Conceito de commit.

Apresentar a Aula 04.
