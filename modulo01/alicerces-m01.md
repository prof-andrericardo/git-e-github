# 🚀 AI01 – Construindo os Alicerces do Manual

## 📚 Disciplina

Versionamento de Código com Git e GitHub

## 🎓 Curso

Técnico em Desenvolvimento de Sistemas

## 📖 Módulo 1 – Fundamentos do Versionamento

## ⏱️ Tempo Estimado

40 minutos

## 🎯 Valor Pedagógico

Esta atividade integradora tem como objetivo consolidar os conhecimentos desenvolvidos nas aulas:

* Aula 01 – Por que o Git Existe?
* Aula 02 – GitHub e o Mercado de Trabalho
* Aula 03 – Markdown, README.md e Criação do Manual

---

# 🎯 Objetivos da Atividade

Ao concluir esta atividade você deverá ser capaz de:

✅ Explicar o que é versionamento.

✅ Diferenciar Git e GitHub.

✅ Compreender a importância da documentação.

✅ Criar e organizar um README.

✅ Estruturar corretamente o Projeto Integrador.

✅ Relacionar GitHub ao mercado de trabalho.

---

# 📖 Situação-Problema

Você foi contratado como estagiário em uma empresa de tecnologia.

Seu gestor pediu que você criasse um repositório inicial para registrar seus estudos e sua evolução profissional.

Esse repositório será utilizado futuramente como:

* material de consulta;
* documentação de estudos;
* portfólio profissional.

Sua missão é construir a base desse projeto.

---

# 🛠️ Parte 1 – Estrutura do Projeto

Crie a seguinte estrutura local:

```text
manual-git-github/
│
├── README.md
│
├── aula01/
│   └── anotacoes.md
│
├── aula02/
│   └── resumo.md
│
└── aula03/
    └── resumo.md
```

---

# 📝 Parte 2 – Construção do README

No arquivo README.md insira:

## Título

```markdown
# Meu Manual de Git e GitHub
```

---

## Autor

```markdown
## 👨‍🎓 Autor

Seu Nome
```

---

## Objetivo

Explique com suas palavras:

> Por que você está criando este manual?

Mínimo:

```text
5 linhas
```

---

## Conteúdo

Liste:

```markdown
## 📚 Conteúdo

- Aula 01 – Por que o Git Existe?
- Aula 02 – GitHub e o Mercado de Trabalho
- Aula 03 – Markdown, README.md e Criação do Manual
```

---

# 📚 Parte 3 – Produção dos Resumos

Produza um resumo para cada aula.

---

## Aula 01

Responder:

### O que é versionamento?

### O que é rastreabilidade?

### Por que o Git foi criado?

---

## Aula 02

Responder:

### O que é GitHub?

### O que é um repositório?

### Como o GitHub pode ajudar na carreira profissional?

---

## Aula 03

Responder:

### O que é Markdown?

### O que é um README?

### Por que a documentação é importante?

---

# 💼 Parte 4 – Mercado de Trabalho

Crie uma seção chamada:

```markdown
## 💼 Mercado de Trabalho
```

---

Explique:

### Quais profissões utilizam Git e GitHub?

Cite pelo menos:

* 5 profissões.

---

### Qual profissão chamou mais sua atenção?

Justifique.

Mínimo:

```text
5 linhas
```

---

# 🚀 Parte 5 – Reflexão Final

Crie uma seção chamada:

```markdown
## 🚀 Minha Reflexão
```

---

Responda:

### O que você aprendeu durante o Módulo 1?

### Qual foi o conteúdo mais interessante?

### Como esse conhecimento pode ajudar sua carreira?

Mínimo:

```text
10 linhas
```
